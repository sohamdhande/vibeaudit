# Security Policy

## ⚠️ Authorized Testing Only Warning

VibeAudit is a security scanning tool designed to simulate attacks (specifically BOLA/IDOR exploitation) on web applications to help identify security vulnerabilities. It must only be used on systems you own or are explicitly authorized to test. Unauthorized scanning or testing of third-party systems is illegal and violates the terms of use of this tool. The authors and maintainers assume no liability for misuse of VibeAudit.

## Reporting a Vulnerability

We take the security of VibeAudit seriously. If you discover a security vulnerability in this project, please follow our responsible disclosure process to report it.

### How to Report

Please report security issues using **GitHub Issues** in the repository:
1. Go to the Issues tab of the repository.
2. Open a new issue and apply the label **`security`**.
3. Provide a clear description of the vulnerability, including step-by-step instructions or proof-of-concept (PoC) code to reproduce the issue.

*Note: Please do not disclose vulnerabilities publicly in the issue description or comments until we have had an opportunity to address them.*

## What Happens Next

After a report is submitted:
1. The maintainers will acknowledge receipt of the report within 48 hours.
2. We will analyze the report, confirm the vulnerability, and coordinate a fix.
3. A security advisory and patched release will be prepared.
4. We will coordinate a publication date for disclosure of the issue alongside the release of the patch.
